# MVP仕様書（Minimum Viable Product）
プロダクト名: Namida（仮）(ピカソの「泣く女」をイメージ)

---

## 1. MVPの目的

- ユーザーの「恥OS」を迂回し、  
  **衝動 → 驚き → 自律** のループを最低限の機能で成立させる。
- 評価される恐怖を避け、出しやすい UI / 快速な生成体験を提供する。
- シェア機能やSNS設計は削ぎ落とし、まずは **1人用の心理的エンタメ装置** にする。

---

## 2. MVPで実装する Core Value（必須価値）

### 🔥 必須価値は 3つだけ
1. **衝動を1タップで入力できる**
2. **AIが「画像＋言葉」でユーザーの代わりに語る**
3. **ユーザーだけが見られる MyRoom に保存される**

この 3つの体験が動けば MVPは成立。

---

## 3. 画面仕様（MVP版）

---

## 3.1 Onboarding（最短構成）
目的: 「ここは評価が存在しない世界」だと伝える

- Slide1: 「ここでは誰もあなたを評価しない。」
- Slide2: 「画像があなたの衝動を代わりに語る。」
- Slide3: 最初の衝動入力（5〜20字）

→ スキップ可  
→ Homeへ

---

## 3.2 Home

### 要素（MVP簡略版）
- 今日の衝動カード（中央）  
  → Create(Text Mode) に1タップで遷移
- 最近の生成ログ（横スクロール）
- ナビゲーション
  - Home
  - Create
  - MyRoom
  - Settings（最小版）

---

## 3.3 Create（MVPでは Text Mode に一本化）

### Input
- 入力欄（placeholder: 「今日はどんな気持ち？」）
- 送信ボタン（大きめ）

### Processing（UI）
- ステップ表示（簡易でOK）
  - 衝動を読む
  - 驚きをつくる
  - 画像を生成

### Output
- 「画像が語る」テキスト + 画像

（Image Uploadモードとショート衝動モードは v2 へ後回し）

---

## 3.4 Result画面

### 構成
- 生成画像（大）
- 語りテキスト
- 反応ボタン
  - 👍 保存
  - ✨ もう1回（再生成）
  - 🔇 非公開（デフォルト）
- フッターメッセージ  
  「あなただけのログに保存されました」

---

## 3.5 MyRoom
- 保存した画像をカードで表示（日時順）
- カードには：
  - 生成画像
  - 語りテキスト
  - 入力衝動（元テキスト）
- 完全非公開  
- 編集機能なし（削除のみ）

---

## 3.6 Settings（MVP版）

- プライバシー設定（単一項目：すべて非公開）
- キャッシュ削除
- バージョン情報

外部サービス連携はすべて削除（Notion/LINE などはv3以降）

---

# 4. 機能仕様（Functional Specs）

---

## 4.1 衝動入力 → 生成フロー

1. ユーザー入力（テキスト）
2. APIへ送信
3. 画像生成（Stable Diffusion系）
4. テキスト生成（GPT系 → “画像が語る”）
5. DBへ保存
6. Result画面へ遷移

---

## 4.2 保存機能（Storage）

保存データ構造：
```json
{
  "id": "uuid",
  "user_id": "uuid",
  "input_text": "string",
  "generated_text": "string",
  "image_url": "string",
  "created_at": "timestamp"
}

--- 

## 4.3 再生成
- 同じ input_text を再利用

- 画像／テキストは毎回変わる（驚き性)

---

# 5.非機能要件（Non-Functional）
- レスポンス：3〜6秒以内（生成系）
- プライバシー：ユーザーのデータは全て非公開
- ログインしなくても使える（匿名ID発行)

---

# 6.開発スコープ（30日MVP）
- Week 1
    - 画面モック（Figma）
    - API基本設計
    - DBスキーマ作成

- Week 2
    - Front: Onboarding / Home
    - API: 生成リクエスト
    - 画像生成・テキスト生成の接続

- Week 3
    - Create → Result → MyRoom の流れを完成
    - 保存処理

- Week 4
    - Debug
    - UX調整
    - 最低限のデプロイ
    - 内部テスト版配布

# 7.MVP後の拡張（将来）
    - 画像アップロードモード
    - ショート衝動モード
    - 驚きアルゴリズムの強化
    - OS診断（恥OS可視化）

## 8.完了条件（Definition of Done）
    - 衝動 → 画像の語り → 保存 の一連が
    - “ストレスなく 10秒以内” に成立する
    - ユーザーが 入力1行で驚きと肯定 を体験できる
    - MyRoomで振り返りができる
- 上記3条件を満たしたら MVP完成。
