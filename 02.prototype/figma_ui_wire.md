## 2.画面定義 (Wireframe Level)

# 01.Onboarding（恥OSの解除）
    - 目的:
        - 「評価されない環境である」ことを理解してもらう
        - ユーザーの不安を軽減し、衝動を出しやすくする
    
    - 画面構成:
    
    - 1.Slide1：世界観の提示
        - テキスト:
            - 「ここでは誰もあなたを評価しない。」
        - イラスト: 柔らかいキャラクター or 霧の中の“ナミダ”

    - 2.Slide2：サービス説明
        - 「画像が、あなたに代わって語る。」
        - 「恥のOSはここでは動かない。」

    - 3.Slide3：最初の衝動キャッチ
        - シンプルな一問
            > 「最近、吐き出したい衝動は？」
        - テキスト入力（5〜20字で十分）

    - 4.登録 or スキップ
        - 「登録して続ける」
        - 「アカウントなしで続ける」

# 02.Home（衝動の入り口）
    - 目的:
        - 常に「ここに書いていいんだ」と感じる
        - 1クリックで“衝動のとりあえず入力”ができる

    - UI構成:

    ----------------------------------
    ① 今日の衝動カード（中央）
　  「何か書きたい衝動はある？」
        → “Create”へ
    ----------------------------------
    ② 画像で作るモード（左）
    ③ テキスト衝動モード（右）
    ----------------------------------
    ④ 最近の生成ログ（横スクロール）
    ----------------------------------
    ⑤ ナビゲーションバー
        - Home
        - Create
        - MyRoom
        - Settings
    ----------------------------------

# 03.Create画面（核となる操作）
    - 3つのモードを用意：

    - ① Text → Image会話モード（メイン機能）

    ----------------------------------
    入力欄：「今日はどんな気持ち？」
    [送信]
    ↓
    生成プロセス表示（3ステップ）
　  ① 衝動の抽出
　  ② 驚きの生成
　  ③ あなたの代弁画像を生成
    ↓
    画像 + テキスト（画像が語る）
    ----------------------------------

    - UIポイント:
        - 入力欄は狭くてOK（恥OSを起動させない）
        - 「送信」ボタンが大きく、優しい

    - ② 画像をアップロードして語らせるモード

    ----------------------------------
    [画像UP]
    ↓
    「この画像はあなたに何と言いたいと思う？」
    ↓
    AIが “画像が語る” 文言を生成
    ----------------------------------

    - UIポイント:
        - 画像に吹き出しが付く
        - 漫画的でも、哲学的でも選べる（トーン切り替え）

    - ③ ショート衝動モード（5秒で完了）
    
    ----------------------------------
    3つの丸ボタン：

    😶　なんとなくモヤモヤ
    🔥　衝動がある
    💧　疲れた

    ↓
    画像 + 言葉が返ってくる
    ----------------------------------

    - UIポイント:
        - 「書けないと先に進めない」にならない設計
        - ワンタップで驚きが生成される

# 04.Result画面（驚き → 自律）
ここが最重要画面。「恥 → 驚き → 自律」を体験する場所。

    - レイアウト:

    -------------------------------
    生成された画像（大）
    -------------------------------
    画像のセリフ or 語り
    「なにも壊れないよ」
    「この衝動、まだ死んでない」
    -------------------------------
    3つの反応ボタン
    👍 保存
    ✨ もう1回驚かせて
    🔇 シェアしない（デフォルト）
    -------------------------------
    “あなただけのログに保存されました”
    -------------------------------

# 05.MyRoom（自分の世界）
ユーザーの作品を時系列で貯めていく。

    -------------------------------
    - カード表示
    - 生成した画像
    - 画像が語ったメッセージ
    - その時の衝動ログ
    - 非公開がデフォルト（SNS文化の逆）
    -------------------------------

    - UIポイント:
        - 「自分の表現は他者評価の前にまず “自分に返ってくる”」
        - を設計思想にする。

# 06.Settings
    - プライバシー（全て非公開がデフォルト）
    - 外部連携（Notion / LINE）
    - キャッシュ削除
    - OS診断（恥OSスコア可視化）

## Figmaでの作業ステップ
このまま反映しやすいように ステップ順に指示 を書く。

    - Step1: フレーム作成
    
    Figmaで以下フレームを一気に作る：
    iPhone 14 Pro（390×844）推奨

    以下を複製：
        - Onboarding x 3
        - Home
        - Create（Text）
        - Create（Image)
        - Create（Short）
        - Result
        - MyRoom
        - Settings

    - Step2: 色と雰囲気
    
    ＊恥 → 驚き → 自律 という遷移を色で表現する

        - Onboarding：淡い灰色（安全領域）
        - Create：淡い青・紫（衝動の揺れ）
        - Result：白 + 暖かいアクセント（肯定）

    - Step3: 基本コンポーネントを作成
        - Large生成ボタン
        - 入力欄（小さめ）
        - 画像表示カード
        - 反応ボタン3種
    コンポーネント化しておくと後が楽。

    - Step4: 主要3フローのプロトタイプ接続
        - Onboarding → Home
        - Home → Create
        - Create → Result
        - Result → MyRoom
    これだけで「体験の核の流れ」が完成する。
