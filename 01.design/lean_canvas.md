## Lean Canvas

# 1.Problem （問題）
    - 1.価値判断を他人に委ね、支配されている人が多い。
    - 2.「おもしろい／つまらない」の外部評価に依存し、創造できない
    - 3.反応が怖くてアウトプットできない

# 2.Customer Segments （顧客)
    - 1.今の笑い・評価文化に疑問を持つが、理由を言語化できない人
    - 2.価値判断に自信がない個人
    - 3.アウトプットが怖いクリエイター予備軍
    - 4.“おもしろさの正解”を探して苦しんでいる人

# 3.Unique Value Proposition（独自の価値）
    - 1.恥・恐怖・衝動を“創造エネルギー”に変換してアウトプットを促す
    - 2.反応の恐怖→対話へ変換するデザイン
    - 3.笑い・驚きを価値判断の再教育ツールとする

# 4.Solution（解法）
    - 1.写真・画像のキャラに話をさせる軽量エンタメメディア
    - 2.ユーザーが“自分の価値判断”で意味づけを行うUI
    - 3.他者評価が不可視化され、反応が怖くない投稿体験
    - 4.驚きを生む “Atom Output（原子的アウトプット）” 生成ツール

# 5.Channels（チャネル）
    - 1.X / 画像SNS（サプライズがバズる構造と相性良）
    - 2.小規模コミュニティ（恥が発生しない安全空間
    - 3.静的1ページサイト（あなたの世界観を提示）
    - 4.Discord（クローズドで育てやすい）

# 6. Revenue Streams（収益モデル）
    - 1.クリエイター向け“価値判断OS”導入コーチング
    - 2.原子的アウトプット生成ツールの月額課金
    - 3.1画像＝1物語の生成エンジン課金
    - 4.コミュニティメンバーシップ
    - 5.世界観・OSライセンスの提供（上級者

# 7.Cost Structure（固定費と変動費）
    - 1.開発（Rust/Serverless/AI API）
    - 2.デザイン（FigJam・Figma）
    - 3.コミュニティ維持
    - 4.広告 or PR
    - 5.コンテンツ制作の時間コスト

# 8.Key Metrics（重要指標）
    - 1.1日の Atom Output 数
    - 2.他者評価ではなく、“自己評価”の変化ログ
    - 3.プロダクト内でユーザーが「価値判断した回数」
    - 4.リピート率
    - 5.サプライズ体験の生成数
    - 6.相手の驚いた反応の回数

# 9.Unfair Advantage（コピー不可能な優位性）
    - 1.「恥」「衝動」「暴力性」「反応の恐怖」—これらを全部“材料”にできる希少性
    - 2.お笑い文化（松本人志的価値観）と内面哲学（スピノザ/バタイユ）を横断
    - 3.驚きを“存在肯定”として扱う感性
    - 4.アウトプットできない苦しみを知る創作者の内側からの設計
    - 5.おもしろい-つまらないに変わる新たな価値を創造する可能性
    - 6.審査型のお笑いとは異なる自律的に価値を判断するシステム
    - 7.お笑い芸人(プロ)-視聴者(素人)の区別を脱するようなプレイヤーの生成
