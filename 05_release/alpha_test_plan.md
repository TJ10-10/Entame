# Alpha Test Plan – Entame(Namida)
最終更新: 2025-12-02  
目的: Entame(Namida) プロジェクトの初期リリース（アルファ版）のテスト計画を定義し、ユーザー体験・機能安定性・Namidaモード遵守を確認する。

---

# 1. テスト対象

1. **OnePane UI**  
   - Web/Next.js / Streamlit / Mobile の表示・操作性  
   - ボタンやコンポーネントの最小ステップ誘導動作

2. **LLM API**  
   - `/api/generate/observation` / `interpretation` / `small_step` の生成精度  
   - Namida モードの維持、生成文の一貫性

3. **Output Prototype**  
   - output_prototype.md の更新・保存・履歴記録  
   - Commit Log / Daily Output 連動

4. **全体フロー**  
   - 「思想 → UI → 行動」の一連の体験  
   - 文脈維持、生成物の最小ステップ化

---

# 2. テスト目標

1. ユーザーが自然に1行行動を完了できること  
2. Namida モードに沿った出力が常に提供されること  
3. UI の最小ステップ誘導が直感的であること  
4. Output Prototype が正しく更新・保存されること  
5. API 応答が安定していること（時間・形式・安全性）

---

# 3. テスト方法

## 3.1 機能テスト
- 各 API エンドポイントを順に呼び出し、レスポンス内容を確認  
- UI ボタン操作に対して最小ステップの行動が誘発されるか確認  
- output_prototype.md の更新後に Commit Log / Daily Output が正しく反映されるか確認

## 3.2 ユーザーテスト
- 社内モニター（3〜5名）にアルファ版を利用してもらい、自然に行動が完了できるか観察  
- 誤操作や迷うポイントをメモして改善案をまとめる

## 3.3 Namida モードチェック
- 生成文・UI 表示・エラーメッセージの文体を Namida モードに従ってチェック  
- 感情表現や語調が過剰になっていないか確認

## 3.4 バグ・改善ログ
- 発見した不具合や改善点を `alpha_test_log.md` に記録  
- 再現手順、画面キャプチャ、改善案を添付

# 5. 成功基準

1. 全ての API が期待通りのレスポンスを返すこと  
2. UI 操作によって1行行動が自然に完了できること  
3. Output Prototype が正常に更新されること  
4. バグは重大度の高いものが 0 件、軽微なものは改善済みまたは修正予定であること  
5. Namida モードの文体・表現が全体で維持されていること

---

# 6. リスク管理

- API 応答の遅延・エラー発生 → ログを記録して再現テスト  
- ユーザー混乱 → UI 誘導改善  
- 文体逸脱 → LLM スタイルガイド再確認・調整  
- 精神的負荷や不適切な生成 → 生成物は創作・行動誘発のみに限定
